================================================================================
  STEP 5: HOW IT WORKS — AUTO-DISPATCH
================================================================================

v3.0 uses an EDITOR REACTOR for auto-dispatch. This means:

  - Commands from Claude are automatically picked up after any AutoCAD action
  - You do NOT need to manually type YQMCP-DISPATCH anymore
  - The reactor fires after every command end, cancel, or LISP evaluation

If auto-dispatch doesn't seem to work, you can always manually trigger:
  Type YQMCP-DISPATCH in AutoCAD command line.

How YQArch commands work via MCP:
  - Claude sends a command (e.g. yq_wall, yq_hole_door)
  - The LISP bridge calls the REAL YQArch function (c:yq_wall), (c:yq_hole_door)
  - YQArch opens its native dialog in AutoCAD
  - You interact with the dialog (set parameters, pick points)
  - The architectural element is drawn properly by YQArch


================================================================================
  STEP 6: USING WITH CLAUDE
================================================================================

Now everything is connected. Here's the workflow:

  1. Open AutoCAD with YQArch loaded
  2. Load the LISP bridge (Step 4) — or it auto-loads from Startup Suite
  3. Open Claude Desktop
  4. Start talking to Claude naturally. Examples:

  -----------------------------------------------------------------------
  YOU:  "Draw walls for a 5m x 4m room"

  Claude calls yq_wall → YQArch wall dialog opens in AutoCAD.
  Set thickness (e.g., 200mm) and draw the walls.
  -----------------------------------------------------------------------

  YOU:  "Add a door on the south wall"

  Claude calls yq_hole_door (shortcut AD) → click on the wall to place
  the door. YQArch automatically cuts the wall and inserts a proper door
  with swing arc.
  -----------------------------------------------------------------------

  YOU:  "Add two windows on the east wall"

  Claude calls yq_hole_win (shortcut AW) → click on the wall to place
  each window. YQArch cuts the wall and inserts proper windows.
  -----------------------------------------------------------------------

  YOU:  "Place rectangular columns at the corners"

  Claude calls yq_r_column (shortcut zzR) → set column size in dialog,
  then place at desired points.
  -----------------------------------------------------------------------

  YOU:  "Add a staircase"

  Claude calls yq_staircase_plan (shortcut LTJ) → YQArch staircase
  dialog opens with all parameters (width, rise, run, etc.)
  -----------------------------------------------------------------------

  YOU:  "Auto-furnish the living room"

  Claude calls yq_autofurniture (shortcut JJ) → YQArch auto-detects
  the room boundaries and places furniture.
  -----------------------------------------------------------------------

  YOU:  "Add bathroom fixtures"

  Claude calls yq_arrangewc (shortcut WC) → YQArch auto-arranges
  toilet, sink, bathtub, etc.
  -----------------------------------------------------------------------

  YOU:  "Dimension everything"

  Claude calls yq_dim_closedspace (shortcut ddZ) → auto-dimensions
  the room/space boundaries.
  -----------------------------------------------------------------------

================================================================================
  TROUBLESHOOTING
================================================================================

PROBLEM: Claude doesn't show autocad-arch tools
  FIX: Restart Claude Desktop. Check config at:
       C:\Users\Admin\AppData\Roaming\Claude\claude_desktop_config.json
       Make sure "autocad-arch" entry exists and path is correct.

PROBLEM: "ACCESS DENIED - License required"
  FIX: Run: node dist/index.js --license
       If not licensed, re-run the owner activation (Step 2).

PROBLEM: Commands sent but nothing happens in AutoCAD
  FIX: 1. Make sure you loaded the LISP bridge (Step 4).
       2. Type YQMCP-DISPATCH in AutoCAD command line.
       3. Check C:\temp\ for .json files.
       4. Make sure YQArch is loaded (YQARCH menu visible in AutoCAD).

PROBLEM: YQArch function not found
  FIX: The YQArch plugin must be loaded BEFORE the LISP bridge.
       Restart AutoCAD, ensure YQARCH menu appears, then load bridge.

PROBLEM: "IPC Error" or timeout
  FIX: Make sure C:\temp\ folder exists.
       Make sure AutoCAD is open with the LISP bridge loaded.

PROBLEM: Need to rebuild after code changes
  FIX: cd "C:\Users\Admin\Desktop\Claude project\yqarch-mcp"
       npm run build
       Restart Claude Desktop.
